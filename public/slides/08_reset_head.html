Let's make a staged mistake .
      
        $ git add my-file.txt
Let's fix it !
      
        $ git reset HEAD my-file.txt

        Unstaged changes after reset:
        M       my-file.txt
What happens if you added the file already? Remember, once a file is added, it is 'staged' in the index. That means Git is aware there has been a change added but not committed.

Let's give it a try. Open your text file again and add some text.

Next, add it to Git using 'git add.'

Whoops! We didn't want to add that. How do we fix it?

Run the next command, 'git reset HEAD my-file.txt.'

You should see a notification in the terminal which says: Unstaged changes after reset: my-file.txt

This command tells Git to unstage the file and set the index of the file back to the last HEAD. HEAD is the name given to the commit from which the working tree's current state was initialized. Your changes still exist in the file but they are no longer staged in the index. Running git status will show the unstaged changes. *INSTRUCTOR: This may take further exposition as HEAD can be a confusing idea for beginners)
