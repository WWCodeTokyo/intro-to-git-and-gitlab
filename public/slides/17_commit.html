Once you have made a change and saved it, go back to the terminal and use 'git status' to see if the changes were registered.

You should see the following output in the terminal, marked in red. This message means that changes have been made but have not been staged for tracking yet.
Stage file for tracking
        
          $ git add README.md
Check status
        
          $ git status
Output
        
          modified: README.md
Let's stage the file for tracking by adding it.

Next, recheck the Git status.

You should see the following output in the terminal, marked in green.
Time to commit!
        
          $ git commit -m "details of the change made"
Check status
        
          $ git status
Output
        
          On branch [BRANCHNAME]
          Your branch is ahead of 'origin/[BRANCHNAME]' by 1 commit.
            (use "git push" to publish your local commits)
          nothing to commit, working tree clean
Commit your change and make sure to write a detailed commit message.

Next, recheck the Git status.

This time, you should see the name of the branch you are on. You should also know that you are ahead of the origin of that branch by one commit. The 'Origin' is the branch on the server. Your local copy contains one change that is not yet on the server copy.
