Let's check the status of our Git Repo.
    
      $ git status
    
  
      
        On branch master

        No commits yet

        nothing to commit (create/copy files and use "git add" to track)
You can run the 'git status' command to check the status of your repo.

*INSTRUCTOR: ask the class to run it and then ask what they see.

*They should see "On branch Master / Initial commit/nothing to commit (create/copy files and use "git add" to track."
Let's create a file and add it to your repo.
    
      $ touch my-file.txt
View the file by running the list command.
      
        $ ls
Tell Git to track our file.
      
        $ git add my-file.txt
To create an empty text file, enter the 'touch' command followed by the name you want to give the file.

Then run the ls command to see that the file was created. Running this command should output the name of the file in the command line.

Now, let's tell Git to track this new file. Do this by typing in the 'git add' command followed by the name of the file you just created. This command tells Git to add it to the tracked files.

*INSTRUCTOR: Tell the class to run the 'git status' command and tell you what they see.(** they should see, Initial commit / Changes to be committed: . . .)

Congrats! Your file is now being tracked by Git!
Now check your repo status
    
      $ git status
    
  
      
        On branch master

        No commits yet

        Changes to be committed:

          new file:   my-file.txt
Time to commit!
    
      $ git commit -m "First commit"
    
  
      
        [master (root-commit) 4315371] First commit
        1 file changed, 0 insertions(+), 0 deletions(-)
        create mode 100644 my-file.txt
Your file is being tracked, but you have not committed the changes to memory. To commit the file, run the 'git commit -m' command. It is important that you add a description of your commit in quotes after the '-m' (m stands for message).

If not, the command line will open VIM. Escaping VIM, the difficulty of escaping VIM that is a standard development joke. (** You may want to cover how to escape VIM ).
