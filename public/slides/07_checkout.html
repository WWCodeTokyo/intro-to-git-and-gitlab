Let's make a local mistake
Nobody's perfect. Let's walk through how to handle mistakes.

Earlier, we created a text file and committed it to be tracked by git. Open that file using notepad/text-edit (right click=> Open with).

Now, type something into the file and save it (ctrl+S).

Whoops! We made a mistake. We didn't mean to change that file. What can we do?!
Let's revert the local changes
      
        $ git checkout my-file.txt
      
    
      
        $ git status
      
    
      
        On branch master
        nothing to commit, working tree clean


In the terminal, type the command 'git checkout my-file.txt.' This command will update the files in your root working directory to match the version in the index.

Go ahead and check it. Close the text file (if still open) and then reopen it. The text you typed in should be gone. But Why?

You made changes to the file but never committed them to the index. Until you commit them, they only exist locally. It is important to remember this as you work with Git. Remember to commit your changes frequently.
