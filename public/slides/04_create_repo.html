First, let's get our bearings.
    
      $ ls
Next, we will make our first local git repository. Before we start, let's make sure we know where we are. Since you have not navigated anywhere, you should be in your root directory. The root directory is usually indicated by the '~' (tilde pronounced til-da)

Run the 'ls' command. This command will list the contents of the directory in which you are currently.
If you are in your root directory
you should see something like this.
If you are not in your root directory, no worries. Run this!
      
        $ cd ~/
Your root directory stores all of the directories that belong under the logged in username. You should see things like 'Desktop,' 'Documents,' 'Downloads,' etc. (**Note, image shows MAC OS directory)

If you are not in your root directory, you can get there by running the 'change directory' command with the tilde.
Make your first Git Repo!
    
      $ mkdir my-first-repo
Move into your new repo
      
        $ cd my-first-repo
Once you are sure you are in your root directory, go ahead and run the following command to make your very first repository *INSTRUCTOR: You may want to explain that it is OK to CD to their Desktop, or another directory to create this folder.

Next, use the 'Change Directory' command to move inside the directory you just made. You must be inside a folder to initialize Git.

Initialize your Git repository by running 'git init.' This command creates an empty Git repository. You should see a notice on the command line telling you. "Initialized empty Git repository in /Usesr/..."

*INSTRUCTOR: A lot is going on behind the scenes here that may be overwhelming for those just learning. A .git directory with the necessary subdirectories and an initial HEAD file is created. In short, this is everything needed to create a master branch and start tracking changes.
Initialize your repository with Git
    
      $ git init
    
  
      
        Initialized empty Git repository in /My/Path/my-first-repo/.git/
