Add your SSH key

https://docs.gitlab.com/ee/ssh/README.html
Before we do anything, we need to set up your SSH Key. What is an SSH Key?

To push your changes to a GitLab server, you need a secure communication channel for sharing information. The SSH protocol provides this security and allows you to authenticate to the GitLab remote server without supplying your username or password each time.

Step by step documentation here: https://docs.gitlab.com/ee/ssh/README.html#doc-nav

Check for an existing SSH key

Git Bash on Windows / GNU/Linux / macOS / PowerShell:
      
        $ cat ~/.ssh/id_rsa.pub
Windows Command Prompt:
      
        $ type %userprofile%\.ssh\id_rsa.pub
Before generating a new SSH key pair, let's check to see if your system already has one.

In your terminal, run the following command (based on your OS).

If you see a string starting with 'SSH-RSA' (pictured here), then you have an existing SSH key pair. You can skip this next part!

Generate a new SSH key pair

Git Bash on Windows / GNU/Linux / macOS / PowerShell:
      
        $ ssh-keygen -t rsa -C "your.email@example.com" -b 4096
      
    
      
        Generating public/private rsa key pair.
        Enter file in which to save the key (/Users/yourname/.ssh/id_rsa):
If you don't see this string, then we need to go ahead and generate an SSH key pair.

Run the keygen command to generate the key pair.

You will see a prompt to enter the file path for where you want to save the file.

Press enter to use the suggested path.

Passphrase

      
        Created directory '/Users/yourname/.ssh'.
        Enter passphrase (empty for no passphrase):
        Enter same passphrase again:
Once you have input a file path, you will be prompted to input a password to secure your SSH key pair. It is a best practice to use a password for an SSH key pair, but it is not required, and you can skip creating a password by pressing enter.

Once you have completed this step, you will see something like this generated.

This funny looking pattern is your SSH Key!

Copy the public SSH key

macOS:
      
        $ pbcopy < ~/.ssh/id_rsa.pub
Windows Command Line:
      
        $ type %userprofile%\.ssh\id_rsa.pub | clip
Git Bash on Windows / Windows PowerShell:
      
        $ cat ~/.ssh/id_rsa.pub | clip
Now, we will need to copy the key so we can add it to our profile on GitLab.

For those of you that already had an SSH key pair, you will need to do these next steps as well.

Add your key to your profile

The final step is to add your public SSH key to GitLab.

Navigate to the 'SSH Keys' tab in your 'Profile Settings.' Paste your key in the 'Key' section and give it a relevant 'Title.' Use an identifiable title like 'Work Laptop - Windows 7' or 'Home MacBook Pro 15'.

If you manually copied your public SSH key make sure you copied the entire key starting with SSH-RSA and ending with your email.
